# Layer 1 — Infrastructure (Dependency Updates) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump every non-Tailwind dependency to its latest stable version, including all majors, while keeping `pnpm validate`, `pnpm build`, and `pnpm test:security` green. No visual or behavioral changes.

**Architecture:** Sequential, grouped bumps. Each group is one commit. After each group, the existing pipeline (`pnpm check && pnpm lint && pnpm build && pnpm test:security`) must pass. Failures are fixed inside the same task (small, scoped patches) — never deferred. Tailwind 3 stays untouched; it moves in Layer 2.

**Tech Stack:** pnpm, SvelteKit 2, Svelte 5, TypeScript 6, Vite 8, ESLint 10, Vitest (not added here — Layer 3), Cloudflare Wrangler 4.90, Firebase 12.13, Groq SDK 1.x, rate-limiter-flexible 11.

---

This layer modifies only:

- `package.json` — version bumps and a single removal (`openai`).

## Task 8: Bump TypeScript (5.8 → 6.0) — BREAKING

**Files:**

- Modify: `package.json` (devDependencies)
- Possibly: `tsconfig.json` — TS 6 may emit deprecation warnings about removed `compilerOptions`.

TypeScript 6 deprecated some flags (e.g., the legacy `importsNotUsedAsValues`) and tightened some inference. Most projects pass clean.

- [ ] **Step 1: Bump TS**

```bash
pnpm add -D typescript@^6.0.3
```

- [ ] **Step 2: Run type check, expect possible breakage**

```bash
pnpm check 2>&1 | head -50
```

If clean, skip to Step 5. If errors appear, capture them and proceed to Step 3.

- [ ] **Step 3: Apply common TS 6 fixes (if needed)**

Three classes of error are likely:

(a) **Stricter `noUnusedParameters` / `noUnusedLocals` in dev deps.** Fix by prefixing with `_`: `function foo(_unused: string)`.

(b) **`isolatedDeclarations` complaints** from new defaults: add explicit return type annotations to exported functions that lack them.

(c) **`tsconfig.json` deprecation warnings.** Open `tsconfig.json`; remove any flag flagged as deprecated in the warning. Do not change `extends` or `compilerOptions.target` unless required.

Apply only the fixes required by reported errors. Do not preemptively retype unaffected code.

- [ ] **Step 4: Re-run type check**

```bash
pnpm check
```

Expected: green.

- [ ] **Step 5: Verify build still works**


Expected: green.

- [ ] **Step 6: Commit**
